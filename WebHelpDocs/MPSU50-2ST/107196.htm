bufferVar.fillmode

This attribute determines if old data is overwritten when the reading buffer is filled.

Type

TSP-Link accessible

Affected by

Where saved

Default value

Attribute (RW)

Yes

Not applicable

See Details

0

Usage

fillMode = bufferVar.fillmode

bufferVar.fillmode = fillMode

fillMode

The reading buffer fill mode, where X is the channel of the module that contains the buffer; set to one of the following:

  • Do not overwrite old data: 0
  • New readings restart at index 1 after acquiring reading at index bufferVar.fillcount: 1

bufferVar

The reading buffer; can be a dynamically allocated user-defined buffer or a default reading buffer

Details

When this attribute is set to 0, the reading buffer does not overwrite readings. If the buffer fills, no more readings are made.

When this attribute is set to 1, new readings are appended to existing readings until the buffer holds bufferVar.fillcount elements. The next reading overwrites the reading at index 1, the following reading overwrites the reading at index 2, and so on.

Example

buffer1 = slot[1].psu[1].makebuffer(100)

buffer1.fillmode = 0

Sets the reading mode of the buffer for channel 1 of the module installed in slot 1 so that it does not overwrite readings.
