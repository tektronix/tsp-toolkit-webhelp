slot[slot].interlock.override

This attribute suppresses or permits interlock errors to be generated.

Type

TSP-Link accessible

Affected by

Where saved

Default value

Attribute (RW)

Yes

Reset
Recall setup

Create configuration script
Save setup

0

Usage

value = slot[slot].interlock.override

slot[slot].interlock.override = value

value

Indicates the state of the interlock override:

  • slot.ON (1)
  • slot.OFF (0)

slot

The slot (1 to 6) containing the card to which the interlock state is applied

Details

This command suppresses errors that would otherwise be generated when the interlock is not closed. If the interlock is not physically connected, channels will still not close.

This attribute exists only for installed cards that support detecting an interlock break. Otherwise, the return value is nil. If the card supports detecting an interlock break, set this attribute to the appropriate response.

To enable interlock override on the card, set to slot.ON. To disable it, set to slot.OFF. This setting applies to all interlocks on the card.

Example

slot[3].interlock.override = slot.ON

Suppresses interlock errors.
